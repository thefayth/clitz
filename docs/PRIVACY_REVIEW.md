# Privacy Review

Review date: 2026-06-19

## Safety Posture

CLITZ treats privacy as a core product feature, especially for missing-person, survivor, provider, shelter, mutual aid, and fundraiser records.

## Public Rendering Rules

- Do not render private contacts.
- Do not render admin-only notes.
- Do not expose exact safe-house addresses.
- Do not expose non-consented survivor locations.
- Default sensitive records to hidden or approximate location.
- Show verification level and source quality.
- Show report and update paths.

## Moderation Rules

- Public submissions land pending.
- Reviews require firsthand/hearsay disclosure.
- Defamation, doxxing, threat, and emergency-claim acknowledgments are required.
- Fraud and abuse reports return to moderation.
- Admin actions create audit records.

## Export Rules

- Public exports exclude private contact fields and admin notes.
- Privileged exports are admin-only.
- CSV imports run validation before write.
- Dry-run summaries and row errors are part of the review trail.

## Live Data Rule

No real sensitive records should be imported or published until legal, moderation, backup, access-control, and human review gates are complete.
